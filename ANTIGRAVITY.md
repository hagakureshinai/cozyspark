# ANTIGRAVITY: Cozy Spark コードベースのリファクタリング

## プロジェクト概要
- **プロジェクト名**: Cozy Spark
- **ゴール**: Astroで作成したサイトのコード整理（リファクタリング）を行い、保守性を向上させる。
- **現在のフェーズ**: 分析 & 戦略選定

## 技術スタック
- **フレームワーク**: Astro 5.0+
- **スタイリング**: カスタムCSS（Scoped + Global）が混在している状態 → *整理方針を決定中*
- **デプロイ**: GitHub Pages (推定)

## 要件定義
現状、作成過程での試行錯誤によりコード（特にCSS）が複雑化し、保守性が著しく低下している。
目標は、コード全体を整理し、スッキリとした状態にすること。

### 提案する戦略（どちらかを選択）
1. **A案：コンポーネント指向 & Scoped CSS（おすすめ）**
   - `Layout.astro` に詰め込まれたヘッダーやフッターを、独立したコンポーネント（部品）として切り出す。
   - 色や文字サイズなどの共通ルールを `global.css` に集約する。
   - 各コンポーネントごとにCSSを記述し、影響範囲を限定する。
   - **メリット**: 直感的でわかりやすい。HTMLが汚れない。標準的なCSSの知識で書ける。
   
2. **B案：Tailwind CSS の導入**
   - 既存のCSSをすべて削除し、ユーティリティクラス（`class="text-red-500"`など）に置き換える。
   - **メリット**: 開発スピードが速い。デザインシステムが統一しやすい。CSSファイルが肥大化しない。
   - **デメリット**: HTMLのクラス名が長くなりごちゃごちゃして見える。Tailwind独自の記法を覚える必要がある。

## MVPスコープ（リファクタリング計画）
- **フェーズ 1: 構造化**
  - ヘッダー・フッターのコンポーネント化。
- **フェーズ 2: デザインルールの統一**
  - カラーパレットや文字サイズの変数を定義し、`global.css` に集約。
- **フェーズ 3: クリーンアップ**
  - 既存ページ（トップページ、作品一覧など）を新しいルールに従って書き直す。
